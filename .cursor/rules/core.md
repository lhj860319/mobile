# 목표
- 실수(회귀/깨짐) 최소화가 최우선.
- 변경은 항상 최소 단위(smallest viable change)로 한다.
- 추측이 생기면, 코드/설정/로그를 먼저 확인하고 진행한다.

# 작업 원칙
- 한 번에 여러 목적(버그/리팩토링/기능)을 섞지 않는다.
- 수정 후 반드시 검증(린트/타입체크/테스트)을 통과시킨다.
- 기존 패턴을 우선한다. 새 패턴 도입은 필요한 경우에만.

# 실행 커맨드(프로젝트에 맞는 것만 남기고 나머지는 삭제)
- npm run lint
- npm run typecheck
- npm run test
- npm run build

# 검증 루틴
1) lint/typecheck
2) 실패 시 로그 근거로 수정
3) test
4) 필요 시 build
5) 전부 통과할 때까지 반복

# 안전 장치
- 파일 삭제/대규모 리네임/대량 포맷팅은 금지(필요 시 최소 범위).
- .env, 키/토큰/자격증명 파일은 절대 컨텍스트에 포함하지 않는다.
